# 空窗监控
## 应用场景
在feed楼层组件中，有时候商品组没有及时配置商品，会导致楼层空窗
## 总结场景
基本上楼层有两种情况，一种是有一个有父元素包裹，此父元素有list属性，只要此父元素没有子元素，则为空。一种是frament元素包裹，相当于没有固定的父元素可以判断，那么这种情况可以换个思路，一般空窗会有一个兜底的组件，带有no-data属性，那么可以通过有此元素判断空窗。
## 具体实现
主要抽象出两个方法：
1. 是判断当前元素的三层元素内无子元素，则视为空

        function isEmptyElement(element) {
            // 检查元素是否带有'xxx-list'属性（兼容a-list和b-list）
            if (!element.matches('[a-list],[b-list]')) return false;

            // 获取元素的三层内所有子元素
            const children = element.querySelectorAll(':scope > *:scope > *:scope > *');

            // 判断三层内是否有子元素
            return children.length === 0;
        }

        // 示例用法
        const elements = document.querySelectorAll('[a-list],[b-list]');
        elements.forEach(el => {
            if (isEmptyElement(el)) {
                console.log('空元素:', el);
            }
        });

2. 通过mutationObserver监听，当前元素中有no-data元素，则视为空

        function observeEmptyElements(target) {
            const observer = new MutationObserver((mutations) => {
                mutations.forEach(mutation => {
                    if (mutation.type === 'childList') {
                        const hasNoData = mutation.addedNodes.some(node => node.nodeType === 1 && node.matches('.no-data'));
                        if (hasNoData) {
                            console.log('检测到空元素:', target);
                            // 可在此处执行空元素处理逻辑
                        }
                    }
                });
            });

            observer.observe(target, {
                childList: true,
                subtree: true
            });
        }
