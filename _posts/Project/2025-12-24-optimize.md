# Debounce 和 Throttle 的区别

在前端开发中，特别是在处理用户输入、窗口调整大小、滚动等频繁触发的事件时，性能优化变得尤为重要。debounce（防抖）和 throttle（节流）是两种常用的技术，用于控制事件处理的频率，从而提高应用的性能和响应能力。下面将详细介绍这两种技术的区别及其应用场景。

## 1. Debounce（防抖）

概念： Debounce 的核心思想是延迟执行。它会在最后一次事件触发后的一段时间内，如果不再有其他事件触发，则执行一次回调函数。如果在该时间段内再次触发了事件，计时器会重新计时。

实现原理：

设置一个定时器。
在事件触发时，先清除之前的定时器（如果存在）。
重新设置一个新的定时器，在指定的时间间隔后执行回调函数。
代码示例：

        1. 函数首先传入两个参数，一是fn回调函数，而是delay延迟时间
        2. 定时器首先设置为null，return一个函数，清理定时器，
        3. 重新创建一个定时器，并且在延迟时间执行回调函数。
            function deBounce(fn, delay) {
                let timer = null;
                return function (...args) {
                    clearTimeout(timer);
                    timer = setTimeout(() => {
                        fn.apply(this, args)
                    }, delay)
                }
            }

应用场景：
搜索框的自动完成建议。
表单验证（在用户停止输入后进行验证）。
按钮点击事件的防重复提交。


## Throttle（节流）
概念： Throttle 的核心思想是按一定的频率执行。它会确保在一定时间内只执行一次回调函数，不管在这段时间内事件被触发了多少次。

实现原理：

记录上一次执行回调的时间。
每次事件触发时，计算当前时间与上次执行时间的差值。
如果差值大于或等于设定的时间间隔，则执行回调函数，并更新上次执行时间。
如果差值小于设定的时间间隔，则不执行回调函数。
代码示例：
1. 设置一个flag，不需要初始值
2. return 一个函数，如果!isThtottle，将flag设置为true，并且执行函数
3. 经过delay时间后，执行setTimeout函数将flag设置为false

function throttle(fn, delay) {
    let isThtottle;
    return function(...args) {
        if (!isThtottle) {
            isThtottle = true;
            fn.apply(this, args)
            setTimeout(() => {
                isThtottle = false
            }, delay)
        }
    }
}

应用场景：

滚动条事件监听（如懒加载图片）。
调整浏览器窗口大小时的重绘操作。
鼠标移动事件的实时反馈（如拖拽操作的平滑性）。
总结
Debounce 适用于需要“合并”多次连续触发事件的场景，只在最后一次事件触发后的指定时间内无其他事件触发时才执行回调函数。
Throttle 适用于需要“节流”的场景，即在一定时间间隔内只允许执行一次回调函数，无论这段时间内事件被触发了多少次。
通过理解并使用 debounce 和 throttle 技术，可以有效地提升前端应用的性能和用户体验。
