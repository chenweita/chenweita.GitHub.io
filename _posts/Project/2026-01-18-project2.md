为什么要使用比较复杂的构建工具？
html css js 开发环境，生产环境基本没有区别
需要我们自己管理标签的加载顺序，控制页面中的位置

工程化发展：
2009 node 预处理工作

标志着开发环境和生产环境 **隔离，管理**
- babel ts
- less sass
- gulp grunt

如何有效管理工具？
webpack
在webpack中所有都是模块

有必要学webpack嘛？


webpack：核心包 包含了webpack构建过程中所有重要的api
webpack-cli：命令行工具 提供了一些命令帮助使用webpack 进行构建

package.json 简单的"script"
"build": "webpack --mode=development --config webpabk.config.js",

webpack.config.js
module.exports = {
    mode: 'development',
    entry: 'src/index.js',
    output: {
        filename: 'bundle.js',
        clean:true
    }
}

生成的bundle是立即执行函数  39：00

entry：入口文件
- stirng 单入口
- array 多入口
- object 多入口

hash：
chunkhash
contenthash

publicPath:输出资源路径前缀

library：库的名称
module：决定了如何处理项目中不同类型的模块 

Resolve：模块解析配置

extension
手写loder：
webpack加载资源类似一个工作管道，在这个过程中一次使用loader对资源进行处理
最终管道结束必须是一个标准的js代码字符串
手写plugin：
增强webpack在自动化构建方面的能力
钩子
希望开发插件能添加一个资源文件列表的功能
1. 输出一个md文件
2. 需要显示打包文件数量和文件的list

compliation对象 此次运行打包的上下文 包含了此次打包的所有信息

webpack构建流程
1. 初始化阶段
    - 初始化参数
    - 创建编译对象 创建compiler对象
    - 初始化编译环境
    - 开始编译 compiler对象run方法，创建comlpliation对象
    - 确认入口
2. 构建阶段
    - 编译模块 entry 调用loader js解析器 转换为ast 递归
    - 完成模块编译 依赖关系图
3. 封装阶段
    - 合并chunk 合并所有代码块 生成最终的代码
    - 优化chunk 优化代码块 例如代码压缩，代码分割
    - 写入文件系统
